Что такое агент?

Когда ты пишешь мне вопрос и я отвечаю — это обычный разговор. Я получаю текст, думаю, возвращаю текст. Один шаг.

Агент — это Claude, которому дали инструменты и разрешили делать несколько шагов самостоятельно.

Вместо одного ответа агент может: прочитать файл → подумать → открыть другой файл → подумать ещё → написать результат. И так столько шагов, сколько нужно.

Аналогия. Обычный запрос — это как спросить коллегу вопрос и получить ответ. Агент — это как дать коллеге задание, ключи от офиса и сказать «разберись и доложи». Он сам решает какие шкафы открыть, что прочитать, что сделать.

Что произошло сегодня

Когда ты спросила про PowerShell permissions, я не отвечал из головы. Я запустил отдельного агента — claude-code-guide — с чистым контекстом и конкретной задачей: найти документацию. Он поискал, вернул результат, я передала тебе ответ.

Ты → я (главный агент) │ └──► claude-code-guide (subagent) │ поиск документации └──► результат → обратно мне → тебе

То же самое с настройкой statusline — там работал агент statusline-setup. Ты видела один ответ, но за ним был отдельный Claude с отдельной задачей.

Зачем это нужно

Три причины запускать агента вместо просто спросить:

1. Задача многошаговая. Агент может сам решать что делать дальше, не спрашивая каждый раз.

2. Задача параллельная. Можно запустить несколько агентов одновременно — каждый делает своё.

3. Контекст нужно защитить. Агент работает в отдельном контексте — не засоряет главный разговор промежуточными результатами.

Проверь себя

Сегодня я запустила агента claude-code-guide. Почему именно агента, а не просто ответила из памяти?
