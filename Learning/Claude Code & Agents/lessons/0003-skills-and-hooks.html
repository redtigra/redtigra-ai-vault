Урок 3 · Claude Code & Agents

Скилы и хуки

← Урок 2: Инструменты агента

Мы разобрались с инструментами — тем, что агент умеет делать. Теперь разберём как настроить поведение агента под себя, не написав ни строчки кода. Для этого в Claude Code есть два механизма: скилы и хуки.

Скил (Skill)

Ты запускаешь вручную командой /название.
Даёт Claude инструкции — как специализированный брифинг.

Хук (Hook)

Запускается автоматически по событию.
Выполняет команду в терминале — без участия Claude.

Аналогия. Скил — это горячая клавиша: ты нажимаешь, запускается заготовленный сценарий. Хук — это датчик движения: он сам срабатывает когда что-то происходит, без твоего участия.

Скилы: заготовленные инструкции

Скил — это обычный текстовый файл с инструкциями для Claude. Когда ты пишешь /teach, Claude читает файл teach.md и следует тому, что там написано. Ты уже используешь несколько скилов каждую сессию:

Скилы в этом проекте:
/teach → открывает обучающую сессию
/process-finance → обрабатывает финансовые документы
/code-review → проверяет изменения в коде
/handoff → готовит передачу задачи другому агенту

Файлы скилов лежат в папке .claude/skills/ — либо глобально (для всех проектов), либо в конкретном репозитории (только для него). Внутри — обычный Markdown: что делать, в каком порядке, что проверить.

Пример: фрагмент файла teach.md
# Skill: teach

Читай MISSION.md и learning-records/,
чтобы понять где мы остановились.
Определи зону ближайшего развития.
Создай урок в lessons/ и открой его.

Именно это произошло сегодня: ты написала /teach, я прочитал скил, затем прочитал твои файлы, подготовил урок. Скил дал мне контекст — я не угадывал что делать.

Хуки: автоматические реакции

Хук — это команда, которая запускается когда происходит определённое событие в Claude Code. Хуки настраиваются в файле settings.json — это делает скил /update-config.

Четыре момента когда может сработать хук:

  • PreToolUse Перед тем как агент использует инструмент. Можно проверить — или заблокировать.
  • PostToolUse После того как инструмент отработал. Например: автосохранение после каждого Edit.
  • Notification Когда Claude Code хочет тебя уведомить. Можно добавить звук или всплывашку.
  • Stop Когда Claude заканчивает работу. Например: git commit после завершения задачи.

Важно. Хук выполняет команду сам терминал — не Claude. Claude не знает что хук существует. Это значит: хуки надёжны и не зависят от того, что Claude решит сделать.

Когда что использовать

Скил — когда...

Ты хочешь дать Claude специализированные инструкции для повторяющейся задачи. Запускаешь вручную, когда нужно.

Хук — когда...

Ты хочешь чтобы что-то случалось всегда при определённом событии — автоматически, без напоминания.

Проверь себя

Ты пишешь /process-finance. Что происходит в этот момент?

Ты хочешь чтобы после каждой завершённой задачи Claude Code воспроизводил звук. Что для этого нужно?
