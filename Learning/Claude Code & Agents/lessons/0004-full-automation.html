Урок 4 · Claude Code & Agents

Полная автоматизация

← Урок 3: Скилы и хуки

Сейчас: ты кидаешь файл в папку → вручную пишешь /process-finance → ждёшь. Цель: ты кидаешь файл → всё остальное происходит само.

Для этого нужны два компонента которые работают независимо друг от друга.

Два слоя автоматизации

Слой 1 — Снаружи Claude: файловый наблюдатель
Отдельный процесс, который смотрит на папку. Не Claude — операционная система. Когда в папке появляется новый файл, наблюдатель это замечает и что-то делает.
Слой 2 — Внутри Claude: headless-режим
Claude Code умеет работать без тебя — запускается из командной строки, выполняет задачу и закрывается. Никакого чата, никакого ожидания ответа. Это называется headless (без головы — без интерфейса).

Аналогия. Слой 1 — это датчик на двери склада: заметил посылку, подал сигнал. Слой 2 — это курьер которого вызвали по сигналу: пришёл, сделал работу, ушёл. Датчик не знает про курьера. Курьер не знает про датчик. Они связаны одним сигналом.

Полный поток

Ты Кладёшь счёт в папку /inbox/
↓
Наблюдатель Замечает новый файл. Запускает Claude Code через командную строку.
↓
Claude Code Запускается в headless-режиме. Читает задачу, выполняет process-finance, закрывается.
↓
Хук Stop Проверяет папку /processed/. Находит свежий файл → показывает уведомление.
↓
Ты Получаешь уведомление. Счёт обработан, лог записан.

Как выглядит headless-запуск

Claude Code можно вызвать из терминала одной строкой. Наблюдатель делает именно это:

# Наблюдатель запускает это когда видит новый файл:
claude --print "В папке /inbox/ появился новый счёт. Обработай его по process-finance."

--print — это флаг который говорит: «не открывай чат, просто выполни и выведи результат». Claude читает CLAUDE.md, находит скил process-finance, следует инструкциям, закрывается.

Как выглядит наблюдатель

На Windows наблюдатель — это PowerShell-скрипт с FileSystemWatcher. Ты запускаешь его один раз, он работает в фоне и ждёт событий.

# Упрощённая схема скрипта (не для запуска — для понимания):

наблюдать папку C:\inbox\
когда появляется новый файл:
    подождать 2 секунды (файл должен полностью скопироваться)
    запустить: claude --print "обработай новый счёт"

Важно. Наблюдатель — это отдельная программа, не Claude. Она написана один раз, запускается при старте компьютера и работает сама. Claude об этом не знает — его просто вызывают как инструмент.

Что ты контролируешь

У каждого слоя своя точка контроля:

Наблюдатель — папка и условие
Какую папку смотреть. Какие файлы считать триггером (только PDF? любые?). Сколько ждать перед запуском.
Claude headless — задача и скил
Что написано в промпте. Какой скил выполняется. Куда пишет результаты.
Хук Stop — уведомление
Проверяет что работа реально случилась. Показывает уведомление только тогда.
